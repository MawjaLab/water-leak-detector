# Agent Operating Rules

## Scope

The repository root is the project boundary.

This project belongs to the **MawjaLab** brand. The ancestor `mawjalab/` brand workspace contains the binding brand definition (`CLAUDE.md`, `brand/`, `.claude/skills/`). Read-only access to that folder is allowed and required for brand-relevant work; base such work on those predefined files. Copy needed brand assets into this repository — never reference paths outside it.

## Prohibited actions

- Accessing sibling repositories (read-only access to the ancestor MawjaLab brand folder is the sole exception)
- Creating dependencies on parent or sibling paths
- Inventing measurements
- Deleting editable CAD source files
- Committing credentials or private keys
- Declaring hardware validation without evidence

## Verification terminology

- Planned: not executed
- Calculated: derived analytically
- Simulated: checked with a simulation
- Inspected: reviewed structurally or visually
- Measured: checked on physical hardware
- Validated: compared successfully with documented requirements
